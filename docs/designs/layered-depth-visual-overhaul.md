## Vision

### 10x Check
A weather experience engine where the app doesn't just show data — it makes you feel the weather. Reykjavik at 11pm = dark indigo with aurora waves. Miami = blazing orange sunset with heat shimmer. Typography shifts weight with temperature. Glass cards tint to match the sky. Every pixel responds to weather data. The loading state is a dawn breaking. The error state is foggy "lost signal." Not a dashboard — an atmosphere simulator.

### Platonic Ideal
Opening the app feels like a portal to another city's sky. Location transitions feel like teleportation — cinematic dissolves. Every pixel responds to weather data. The loading state is dawn breaking. Even error states feel intentional. A new engineer joining in 6 months sees the three-layer architecture and thinks "oh, that's clever and obvious at the same time."

## Accepted Scope (added to this plan)
- Temperature-responsive typography (font weight mapped to temperature range)
- Glass card sky tinting (CSS custom property from sky gradient → .glass background)
- Animated loading state (dawn-breaking gradient replacing spinner)
- Parallax depth on scroll (background layers move at different speeds)
- Time-lapse mode (24-hour gradient cycle in ~10 seconds)

## Additional Scope (from CEO Review)
- Full weather scene library (~15 WMO conditions, not just initial 4-6)
- Debug overlay (dev-mode, keyboard toggle, shows FPS/particles/perf level)
- Canvas visibility change handling (pause on tab hide)
- README demo GIF/video recording
- Parallax desktop-only (disabled on mobile < 768px)
- 3 performance levels (not 4): reduced-motion / CSS-only / CSS+Canvas

## Implementation Decisions
- **Canvas:** Raw Canvas 2D API (no library) — portfolio signal
- **Strategy:** Progressive Foundation (sky gradients → 2 scenes → expand)
- **Parallax:** JS scroll + rAF (not CSS perspective)
